Cara mapping IP address ke domain menggunakan file hosts di Windows

Apa itu Local DNS?

Local DNS adalah cara untuk memetakan (mapping) IP address ke nama domain secara lokal di komputer, tanpa perlu mengubah DNS publik. Ini dilakukan dengan mengedit file hosts di Windows.

Kegunaan

Berguna untuk mengakses server internal, testing, atau bypass DNS publik untuk domain tertentu.

1

Buka File Hosts sebagai Administrator

File hosts terletak di folder system Windows. Untuk mengeditnya, kamu perlu membuka dengan hak Administrator.

Lokasi file hosts:

Path 
C:\Windows\System32\drivers\etc\hosts

Cara membuka dengan Notepad (Administrator):

  1. Tekan Win + S, ketik Notepad
  2. Klik kanan pada Notepad â†’ Run as administrator
  3. Di Notepad, klik File â†’ Open
  4. Navigate ke path di atas
  5. Ubah filter file dari "Text Documents" ke All Files (*.*)
  6. Pilih file hosts dan klik Open
Penting!

Harus Run as Administrator, jika tidak perubahan tidak bisa disimpan.

2

Tambahkan Entry DNS

Di bagian bawah file hosts, tambahkan baris baru dengan format:

Format 
# Format: IP_ADDRESS    DOMAIN_NAME
IP_ADDRESS    domain.example.com

Contoh entry untuk SD-WAN:

hosts 
36.94.252.50	pemkotmks.sdwanmt1-1.telkom.co.id
Tips

Gunakan TAB atau beberapa spasi untuk memisahkan IP dan domain. Satu baris = satu mapping.

3

Simpan File

Setelah menambahkan entry, simpan file:

  1. Tekan Ctrl + S atau klik File â†’ Save
  2. Pastikan tidak ada error saat menyimpan
Berhasil!

Jika file tersimpan tanpa error, konfigurasi DNS lokal sudah aktif.

Flush DNS Cache (opsional)

Jika perubahan tidak langsung berlaku, buka CMD dan jalankan: ipconfig /flushdns

4

Buka Browser

Buka browser untuk mengakses domain yang sudah di-mapping.

Rekomendasi

Gunakan Firefox karena memiliki network settings terpisah dan bisa dikombinasikan dengan FoxyProxy.

5

Akses Domain

Ketik URL yang sudah di-mapping di address bar browser:

URL 
https://pemkotmks.sdwanmt1-1.telkom.co.id

Browser akan mengarahkan request ke IP 36.94.252.50 berdasarkan mapping di file hosts.

Selesai!

Kamu sekarang bisa mengakses domain menggunakan DNS lokal.

Troubleshooting

Jika domain tidak bisa diakses:

  • Pastikan file hosts tersimpan dengan benar (bukan .txt)
  • Jalankan ipconfig /flushdns di Command Prompt
  • Restart browser
  • Pastikan IP address dan domain tidak typo
  • Cek koneksi ke IP dengan ping 36.94.252.50
CMD 
# Flush DNS cache
ipconfig /flushdns

# Test ping ke IP
ping 36.94.252.50

# Test resolve domain
nslookup pemkotmks.sdwanmt1-1.telkom.co.id
